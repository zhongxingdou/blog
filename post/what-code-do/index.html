代码在干什么？

2020-04-16

代码有长有短，
如 windows 源码，
长到这辈子可能都看不完。
如 print(“hello world”)，
短到一行也可以运行。
或长或短，
总归是写出来有用的，
那代码到底都干了啥呢？

经某人总结，
代码干了一件事：
“找出东西来用”。
这挺好理解，
上街得坐车。
做饭得用锅，
但是，它并不简单。

做饭得放米，
米在米缸里，
放米得看几张口，
量了米得洗，
洗完米再加水，
加少了磕牙，加多了变糊，
1x1.5 是瘦杆米，
1x1 是胖墩米，
擦干手来插上电，
最后记得按对钮。

其实呢，一切都是数据，
米是1，缸是2，锅是3，水是4，
放米a，洗米b，加水c，插电d，按钮e，
其实呢，一切变化都是数据变化，
电流要跑多少回，水要蒸腾几朵云，
才能把生米给做成了熟饭。

So，“找出东西来用” 就是，
编码数据，
写入数据，
取出数据，
计算数据。
So，你学会做饭了吗？
