GraphQL 学习笔记

2020-05-21

GraphQL 是基于图理论的一种用于查询与修改数据的协议

基本类型与表达式

  • String "hello"
  • Float 8888.88
  • Boolean true false
  • ID
  • enum Animal { Monkey, Mouse }
  • 参数 $var
  • 数组 [item]
  • 不可空类型 Photo! ，值不能为 null
  • 不可空数组 [item]!，数组至少有1项才合法
  • [item!]!，数组至少有1项，数组成员不能为 null
  • 标量 scalar DateTime，用于扩展基本类型
  • union SchoolPeople = Teacher | Student，表示类型的值可以是提供的类型其中之一
  • ...on Teacher { course }，引用类型的部分成员

类型

定义类型

type User {
    name: String!
    gender: Boolean!
    interest: String
}

类型间引用

type User{
​	favorites: [Favorite!]
​	friends: [Friendship!]!
}

type Favorite {
​	name: String!
}

type FriendShip {
​	friend_a: User!
​	friend_b: User!
}

定义接口

与 type 定义的类型类似，但是可被继承（实现）

interface People {
​	name: String!
}

type Worker implements People {
​	name: String!
​	position: String!
}

输入类型

与 type 定义的类型类似，但是专用于参数

input Message {
​	title: String
​	content: String!
}

标量

声明标量

scalar DateTime

实现标量

new GraphQLScalarType {
    name: 'DateTime',
    description: 'A valid date time value.'

    parseValue: value => new Date(value),                  // 查询变量的值将用它转换
    serialize: value => new Date(value).toISOString(),   // 确保返回值有统一的格式

    parseLiteral: ast => ast.value // 查询字面量最先被解析为AST时，可以转换它的值，如果需要
}

查询

query users {
​	aliasName: name
​	friends(genderParam: FEMALE) {
​		name
​		gender
​	}

​	...on work {
​		position
​		office
​	},	

​	...on study {
​		grade
​		school
​	}
}

片段

fragment userInfo on User {
​	name
   gender
}
